# 热点哨兵 (Hot Sentinel) - 需求文档

## 项目愿景

小小作为一名 AI 编程博主，需要第一时间获取 AI 大模型等前沿热点变化，不能依赖人工手动搜索，而是利用工具自动发现指定热点变化，并及时发送通知，走在吃瓜第一线。

## 功能需求

### 功能一：仪表盘 (Dashboard)

- 数据总览：展示热点总数、活跃关键词数、未读通知数、AI 分析覆盖率等核心指标（数字动画效果）
- 近 7 天趋势图：ECharts 折线图展示每日热点采集量变化
- 数据源分布：ECharts 环形饼图展示各数据源采集热点占比
- 24 小时采集量：ECharts 柱状图展示每小时的采集量分布
- 重要性分布：ECharts 雷达图展示不同重要性等级的热点数量
- 关键词活跃度排行 TOP 10：按关联热点数量排序，可视化进度条展示

### 功能二：关键词监控

- 用户可以手动输入关键词，支持为每个关键词设置独立的采集频率（1/5/15/30/60分钟 及 12/24小时，或使用全局默认值）
- 支持添加、编辑、启停、删除关键词
- 系统定时从多个信息源搜索关键词相关内容
- 利用 AI 识别内容是否真正与关键词相关，过滤假冒/低质/付费内容，评估可信度和重要性
- 一旦发现真实相关内容，第一时间发送通知给用户（浏览器通知 + WebSocket 实时推送 + 邮件）
- 支持一键导出单个关键词的监控报告为 Excel（含标题/摘要/来源/重要性/可信度等字段）
- 监控概览面板：展示活跃关键词数、总计关键词数、今日已检查数、当前 AI 引擎信息

### 功能三：热点发现

- 用户可输入关键词搜索已有的热点数据（支持标题/内容/范围多字段匹配）
- 支持多维度排序：最新时间、重要性、综合热度、Twitter热度、智能排序
- 支持高级过滤：数据源筛选、重要性阈值、蓝V认证、含图片、含视频、含链接
- 热点卡片展示：标题（可点击外链）、来源标签、范围标签、AI 摘要、可信度进度条、重要性评分、互动数据（点赞/评论/浏览/转发/引用/收藏）、热度趋势（上升/下降/平稳）
- 支持一键导出当前热点列表为 Excel
- 支持热点媒体预览（图片点击放大）

### 功能四：全网搜索

- 用户输入关键词，实时从 16 个数据源并行采集搜索
- 支持选择特定数据源进行定向搜索
- 支持 AI 智能分析排序（判断相关性、评估可信度、生成摘要）
- 搜索结果展示：标题（可点击外链）、AI 分析报告（重要性/可信度/相关性）、内容预览、AI 摘要、来源标签、互动数据、作者信息
- 搜索建议：根据热门关键词自动推荐相关搜索词
- 搜索历史提示：显示搜索耗时、分析结果数、模型信息

### 功能五：通知中心

- 展示所有监控通知：标题、内容摘要、关联热点 ID、来源、重要性、发布时间
- 支持排序：最新时间、重要性优先、未读优先
- 支持过滤：数据源、重要性阈值、已读/未读状态
- 支持标记单条已读 / 全部已读
- 支持一键导出通知为 Excel
- 未读通知数角标实时更新（60 秒轮询 + WebSocket 实时推送）
- WebSocket 连接状态实时显示（在线/离线）
- 浏览器通知开关状态显示

### 功能六：系统设置

- **通知设置**：邮件通知开关、通知邮箱（支持多个，逗号分隔）、浏览器通知开关
- **采集设置**：默认采集频率（1/5/15/30/60分钟 及 12/24小时）
- **AI 设置**：AI 内容审核开关、AI 模型选择（DeepSeek V4 Flash / 讯飞 Spark X / 通义千问 Qwen3.6-Flash）
- **本地 AI**：支持本地 Ollama 服务（Gemma3 4B / Qwen3.5 4B），自动健康检查，不可用时自动降级到云端 API
- **数据源控制**：16 个数据源独立开关（Twitter、Bing、360搜索、B站搜索、百度搜索、微博搜索、搜狗搜索、知乎、掘金、CSDN、开源中国 等），自动同步 `settings` 表控制定时任务启用
- **配置说明**：频率选择建议、AI 审核说明、邮件说明

## 通知方式

1. **浏览器实时推送**：通过 WebSocket（Socket.IO）实时推送通知到所有在线客户端，调用 Web Notification API 弹出系统通知
2. **页面内通知铃铛**：导航栏铃铛图标显示未读数量角标，新通知到达时顶部弹出 Toast 提示
3. **邮件通知**：通过 QQ 邮箱 SMTP（nodemailer）发送邮件通知，支持批量发送多关键词多热点，精美 HTML 邮件模板

## 热点检查频率

| 频率     | 间隔          | 适用场景                         |
| -------- | ------------- | -------------------------------- |
| 极实时   | 每 1 分钟     | API 消耗较大，适合极度重要关键词 |
| 较实时   | 每 5 分钟     | API 消耗较多，适合重要关键词     |
| 平衡选择 | 每 15 分钟    | 推荐默认值                       |
| 省资源   | 每 30 分钟    | API 消耗较少                     |
| 低频     | 每 60 分钟    | 轻度监控                         |
| 按天     | 每 12/24 小时 | 极低频率，适合长期观察           |

> 采集频率通过数据库 `keywords.crawl_frequency` 字段 + `settings.default_crawl_frequency` 配置，支持每个关键词独立设置。

## AI 分析

### 多 AI 后端支持

| AI 后端     | 模型标识               | 说明                           | 使用方式            |
| ----------- | ---------------------- | ------------------------------ | ------------------- |
| DeepSeek    | deepseek-v4-flash      | 默认云端 AI，轻量快速          | OpenAI SDK 兼容 API |
| 讯飞 Spark  | spark-x                | 深度推理模型                   | OpenAI SDK 兼容 API |
| 通义千问    | qwen-turbo             | 平衡型模型，使用 qwen3.6-flash | OpenAI SDK 兼容 API |
| 本地 Ollama | gemma3:4b / qwen3.5:4b | 本地部署，无需网络，隐私安全   | Ollama HTTP API     |

- **自动降级**：本地 Ollama 不可用时自动降级到云端 API
- **Spark/Qwen** 走特殊解析流程（清理 markdown 代码块、提取 JSON）
- 支持在设置中切换 AI 模型（云端模式）或通过 `.env` 配置本地模式

### AI 分析场景

1. **关键词监控审核**：判断爬取内容是否真正与关键词相关、过滤假冒/低质/广告内容、评估重要性和可信度、生成精炼标题和范围标签
2. **全网搜索结果分析**：同关键词审核，额外生成 AI 分析报告（模型/耗时/相关结果数）
3. **热点话题总结**：对采集到的热点内容进行去重合并，提炼核心话题，按重要性排序，生成摘要描述

### AI 可信度评估体系

- 综合评估来源权威性、内容事实依据、标题党检测、互动数据真实性
- 对社交媒体内容（Twitter/微博）特别关注蹭热度营销号、水军刷量检测
- 对搜索引擎结果优先信任官方网站和知名媒体
- 标题含感叹号/问号/诱导性词汇扣减可信度 10-20 分
- 内容明显过时重要性不超过 3 分

## 数据趋势追踪

- 系统每次检查关键词时自动为关联的所有热点创建快照（topic_snapshots 表）
- 记录每个时间点的点赞数、评论数、浏览量、重要性评分
- 前端热点列表展示热度趋势（上升 🔥 / 下降 ❄️ / 平稳 ➡️），基于快照数据计算百分比变化
- 支持查看任意热点的 7 天趋势详情

## 产品形态

1. **响应式 Web 页面**：桌面端 + 移动端均可使用，独特终端黑客风 UI（Terminal Hacker + Cyber Neon）
2. **Agent Skills**：封装为 AI Agent 可调用的技能包（`skills/hot-sentinel-skill.md`）

## UI 设计风格

**主题：终端黑客 + 赛博霓虹 (Terminal Hacker + Cyber Neon)**

- 深色背景（`#06080f`）+ 霓虹绿色/青色/紫色渐变
- 毛玻璃卡片效果（backdrop-filter + rgba 背景）
- 扫描线动画、网格覆盖层、极光背景
- 等宽字体、SVG 内联图标
- 响应式布局（Bento Grid 桌面多栏 + 移动端单栏堆叠）
- 数字滚动动画、骨架屏加载态
- 鼠标跟随光晕效果
